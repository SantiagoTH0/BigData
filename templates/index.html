Gestión de vehículos de emergencia

Coordinación inteligente para respuesta rápida y segura.

Descripción

Este proyecto integra la arquitectura de datos con plataformas de análisis y desarrollo web para optimizar la toma de decisiones logísticas en tiempo real. Nuestro enfoque asegura que los activos críticos (ambulancias, bomberos, patrullas) sean despachados, monitoreados y mantenidos con la máxima eficacia.

Contexto

El sistema que hemos desarrollado aborda un problema muy grave y real en muchas comunidades: la gestión lenta y caótica de los vehículos de emergencia. En las ciudades que no son "inteligentes", la falta de información en tiempo real pone vidas en riesgo. El desempeño se mide con reportes viejos, y nadie sabe realmente dónde está la ambulancia más cercana.

Por eso, nuestro proyecto nace de la necesidad de salvar ese tiempo valioso. Creamos este sistema desde cero, pues, como no teníamos datos reales de una ciudad, simulamos todos los millones de registros de telemetría y eventos para probar nuestro modelo, garantizando que el diseño funciona con la presión de una operación real.

Objetivos

Reducción del TTA (Time to Arrival): Reducir el tiempo promedio que tardan los vehículos en llegar a la escena del incidente, midiendo con precisión la Latencia de Despacho y el Tiempo de Respuesta en Ruta.

Optimización de vehiculos: Mantener un alto porcentaje de vehículos en estado 'Disponible' mediante el análisis predictivo de las alertas de telemetría.

Análisis Geoespacial: Identificar zonas de alta densidad de incidentes mediante cálculos de proximidad (GEOGRAPHY) para redistribuir activos(vehiculos) de manera preventiva.

Componentes clave y datos recopilados

El modelo de datos está diseñado para manejar la complejidad de la infraestructura de emergencia:

- Grupo de Activos: Vehículos de Emergencia (Ambulancia, Patrulla, Carro de Bomberos) con sus respectivas bases (PropiedadesInstalacion).

- Gestión de Turnos: Asignación de Usuarios (personal) a Vehiculos mediante la tabla PersonalVehiculo para mantener la trazabilidad de la tripulación.

- Datos IoT de Alta Frecuencia: La telemetría se segmenta en dos flujos críticos:

  • Ubicación (Series de Tiempo): Registros por segundo (MedicionesUbicacion) para reconstrucción de rutas y cálculos de proximidad.
  • Eventos (Discretos): Alertas de fallas (Batería Baja, Código de Falla, Alerta de Temperatura) registradas en MedicionesTelemetricasEvento.
Arquitectura de plataforma

- Arquitectura de plataforma: La plataforma Flask sirve como portal para el consumidor final, mientras que el análisis es potenciado por SQL Server:

- Ingesta (SQL Server MER): Datos brutos de IoT y transaccionales se almacenan en el Modelo Entidad-Relación para garantizar la integridad de la fuente.

- Transformación (ETL/ELT): Los datos se limpian, se validan y se migran al Data Warehouse (Modelo Estrella/Copo de Nieve) para su análisis.

- Visualización (Power BI): Los indicadores de rendimiento (KPIs) se consumen directamente del DW para generar tableros interactivos que permiten la toma de decisiones en tiempo real.
